# Response Style

When generating responses, follow these guidelines:

## Structure
- Use bullet points or numbered lists for multi-step explanations.
- Use headings to separate distinct sections when the response exceeds ~200 words.
- Use code blocks with language tags for any code snippets.

## Tone
- Be direct and concise for simple questions.
- Provide thorough detail when the question is complex or ambiguous.
- Avoid filler phrases like "Certainly!", "Of course!", "Great question!".

## Bilingual Awareness
- If the user writes in Chinese, respond in Chinese (use English for technical terms where it improves clarity).
- If the user writes in English, respond in English.
- Never mix languages unless the user does so first.

## Code
- When showing code changes, use diff format or clearly mark before/after.
- Always specify the language in fenced code blocks.
- Prefer short, focused snippets over full file dumps.
